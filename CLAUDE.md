# CLAUDE.md

## Project Overview

**async2** — SourceMod C++17 extension: async HTTP/S (curl), JSON (simdjson), MessagePack, TCP, UDP, WebSocket. Static deps in `third_party/`, single libuv event loop thread for all I/O.

## Build

```bash
git submodule update --init --recursive
./build_deps.sh x86_64    # or x86 for 32-bit
./build.sh x86_64          # or x86, or x86,x86_64 for both
```

Manual cmake: `mkdir build-x86_64 && cd build-x86_64 && cmake .. -DSM_PATH=<sourcemod> -DCMAKE_BUILD_TYPE=Release && cmake --build . -j`

Windows: `build_deps.bat` then `cmake .. -A x64 -DSM_PATH=<sourcemod>` + `cmake --build . --config Release`. Uses per-project cmake flags: `-DZLIB_BUILD_SHARED=OFF`, `-DBUILD_SHARED_LIBS=OFF -DBUILD_STATIC_LIBS=ON` (nghttp2), `-DCMAKE_C_FLAGS=/DNGHTTP2_STATICLIB` (curl, needed because curl's `FindNGHTTP2.cmake` doesn't propagate the static define) — NOT standard `BUILD_SHARED_LIBS`.

### Dependencies

Static libraries via git submodules (`third_party/`): libuv, libcurl 8.18.0 (OpenSSL 3.5 LTS on Linux / Schannel on Windows, c-ares), c-ares, zlib, nghttp2, brotli, simdjson, OpenSSL. Header-only: tsl/robin-map, xxHash.

**OpenSSL**: Git submodule in `third_party/openssl`. Built via Perl `Configure` + `make` (not CMake — OpenSSL doesn't support it). `build_deps.sh` runs out-of-source build in `build_tmp/$ARCH/openssl-build/`. Stripped config: `no-shared no-ssl3 no-tls1 no-tls1_1 no-dtls no-engine no-fips no-legacy no-dsa no-dh` plus unused ciphers. For 32-bit: target `linux-x86`; for 64-bit: `linux-x86_64`. `install_sw` installs libs+headers without docs.

## Architecture

### Threading Model

1. Game thread enqueues ops via `LockedQueue` + `uv_async_send`
2. Event thread processes: curl HTTP, TCP, UDP, DNS (c-ares), WebSocket
3. Completed ops pushed to `done_queue` (HTTP) / `socket_done_queue` (TCP/UDP/WS)
4. `OnGameFrame` drains queues, dispatches SourcePawn callbacks

### Critical Invariants

**Close always fires callbacks**: All Close() calls (HTTP, TCP, UDP, WS) fire their callback. HTTP Close() on in-flight → `CURLE_ABORTED_BY_CALLBACK` (42). TCP/UDP close → `bool userClosed`. Stale events after Close() suppressed.

**HTTP auto-close**: Handles auto-freed after callback via `FreeHandle`. Users do NOT call Close() in callbacks (safe no-op, returns 2). Close() only for: (1) freeing unused handles, (2) cancelling in-flight.

**Handle lifecycle**: Two-step close pattern using `MarkHandleClosed` + `FreeHandle`:
1. `MarkHandleClosed(id)` — sets `closed` flag so `Get*` returns nullptr (prevents use-after-close). Handle stays in map for later `FreeHandle`. Use when user/plugin initiates close but async completion hasn't fired yet (HTTP `Native_HttpClose` on in-flight, `OnClientDisconnecting`).
2. `FreeHandle(id)` — deletes object, removes from map, recycles handle number. Use when the handle is fully done: HTTP `OnCompletedGameThread`, socket `*_CLOSED` event dispatch, or immediate close of non-in-flight handles. `FreeHandle` on an already-freed handle is a safe no-op.
Socket close natives (`TcpClose`, `UdpClose`, `WsClose`) set `handle_closed` on the socket object but do NOT call `MarkHandleClosed` — the close event handler calls `FreeHandle` in `DispatchSocketEvent`.

**DataNode per-node refcount**: Each DataNode has `std::atomic<uint32_t> refcount` (starts at 1). `Incref()` takes a new reference; `Decref()` releases one — when refcount hits 0, recursively Decrefs children and returns node to pool. Child handles (`GetObject`/`GetArray`) Incref the child node; when the parent is Decref'd, only nodes with refcount 0 are freed — child subtrees survive independently. This replaces the old `shared_ptr<DataNode>` + orphan mechanism.

**Race guards**: `handle_closed`, `completed`, `in_event_thread` are `std::atomic<bool>`. `OnAsyncAdd` checks `handle_closed` to skip cancelled requests. `OnAsyncCancel` sets `completed = true` to prevent double-push.

**HTTP retry**: `PrepareForRetry()` clears response state but NOT `post_body` while `in_retry_wait` (curl needs the pointer for `CURLOPT_POSTFIELDS`). Cancel during retry: `OnAsyncCancel` stops pending timer, sets `completed=true`. `OnRetryTimer` checks `completed` for timer/cancel race. Retry fields set on game thread before Execute, read on event thread after enqueue — safe via queue lock happens-before.

**`uv_close` before final `uv_run`**: Every uv handle must be `uv_close`'d before the final `uv_run(UV_RUN_DEFAULT)`. That call blocks until all handles closed — any leaked handle hangs the game thread in `epoll_pwait`. Never just `delete` or `uv_*_stop` without `uv_close`.

**Shutdown sequence** (`SDK_OnUnload`): (1) Remove OnGameFrame/listeners, (2) signal+join event thread, (3) close retry timers, (4) close WS handles, (5) close TCP/UDP uv handles, (6) shutdown DNS + curl_multi_cleanup, (7) close remaining uv handles + run loop for close callbacks, (8) uv_loop_close, (9) drain all queues, (10) CleanHandles.

**TCP accept deferred read**: Accepted sockets don't start reading until `ACCEPT_CONFIG` op assigns `handle_id`. Prevents `TCP_DATA` events with `handle_id=0`. `ACCEPT_CONFIG` enqueued before `onAccept` fires, so `TcpSend` inside `onAccept` works.

**WS handshake kick-start**: After `curl_multi_add_handle`, `ProcessWsConnect` must call `curl_multi_socket_action(CURL_SOCKET_TIMEOUT)` + `CheckCompletedJobs`. Timer callback alone unreliable.

**WS completed handles**: Don't call `curl_multi_remove_handle` on successful WS CONNECT_ONLY handles (must stay in multi). Track in `ws_curl_handles_` set.

**WsCleanup never pushes events**: All paths must push `WS_CLOSED` event before calling `WsCleanup`.

**simdjson error handling**: Never apply `!` or implicit bool to `simdjson_result<T>` — always `.error()` first, then `.value()`. For `is_null()`, check error before value (input starting with `n` but not `null` → `INCORRECT_TYPE`).

### DNS

Two independent subsystems that cannot share state:
- **HTTP**: curl's internal c-ares channel (`-DENABLE_ARES=ON`). Curl manages its own DNS cache (60s default, `CURLOPT_DNS_CACHE_TIMEOUT`). No API to inject custom resolver.
- **TCP/UDP**: `DnsResolver` class — separate c-ares channel, app-level cache (mutex-protected, configurable TTL). c-ares qcache disabled (`ARES_OPT_QUERY_CACHE` ttl=0) so our flush/stats APIs work. `Reinit()` destroys/recreates channel on event thread for new settings.

### Key Classes

- **`Async2Extension`** (`src/extension.cpp`) — SDK_OnLoad/Unload, OnGameFrame, client tracking (`g_client_handles` reverse index)
- **`EventLoop`** (`src/event_loop.cpp`) — libuv loop + curl_multi + DnsResolver, all I/O, retry timers, pool stats
- **`HttpRequest`** (`src/http_request.cpp`) — per-request state/lifecycle. `PrepareForSend()` (game thread), `SetupCurl()` (event thread: compression + curl setup)
- **`HandleManager`** (`src/handle_manager.cpp`) — seven types: HTTP_REQUEST, JSON_VALUE, TCP_SOCKET, UDP_SOCKET, WS_SOCKET, LINKED_LIST, ITERATOR. `Handle.closed` flag + free list. `MarkHandleClosed` / `FreeHandle` pattern (see Handle lifecycle above)
- **`DataNode`** (`src/data/data_node.h`, `src/data/data_node.cpp`) — tagged union DOM (Null/Bool/Int/Float/String/Array/Object/IntMap/Binary), pool-allocated via `FixedPool` (`src/data/data_node_pool.h`). Per-node `std::atomic<uint32_t> refcount` for lifetime management (jansson model). `Incref()` / `Decref()` — when refcount hits 0, recursively Decref children and return to pool.
- **`DataHandle`** (`src/data/data_handle.cpp`) — thin wrapper: just `DataNode* node`. Destructor calls `Decref(node)`. No shared_ptr, no iterator state.
- **`DataIterator`** (`src/data/data_iterator.h`, `src/data/data_iterator.cpp`) — standalone iterator for Object/IntMap containers. Holds `DataNode*` with Incref'd refcount. Own handle type (`HANDLE_ITERATOR`).
- **`DnsResolver`** (`src/dns_resolver.cpp`) — c-ares + app-level DNS cache
- **`TcpSocket`** (`src/tcp_socket.h`), **`UdpSocket`** (`src/udp_socket.h`) — socket state machines
- **`WsConnection`** (`src/ws_connection.h`) — WebSocket via curl CONNECT_ONLY=2 + uv_poll_t, message reassembly, auto-ping, close handshake
- **`SocketEvent`** (`src/socket_event.h`) — event struct with `MakeSocketEvent`/`MakeErrorEvent` factory helpers
- **`LockedQueue`** (`src/locked_queue.h`) — `std::mutex` + `std::atomic<bool>` for lock-free HasItems()
- **`DataMap`** (`src/data/data_map.h`) — `DataMap` (string-keyed) and `IntMap` (int64-keyed) aliases for `tsl::robin_map` with `xxh3_hash`. Change only this file to swap implementations.

### SourcePawn API

`sourcepawn/async2.inc` — natives and methodmaps grouped by type, curl enums at end. Use `async2_HttpNew()` not `new WebRequest()`.

- `SetBody`/`SetBodyString` and `SetBodyJSON`/`SetBodyMsgPack` are mutually exclusive — each clears the other. `SetBodyJSON`/`SetBodyMsgPack` **consume the JSON handle**: always DeepCopy the tree for safe handoff to the event thread (child nodes may be shared via GetObject/JsonRef). Tree is serialized on the event thread. `Close()` after send is a safe no-op. Same for `WsSendJSON`/`WsSendMsgPack`.
- `WsSendJSON`/`WsSendMsgPack` also **consume the JSON handle** — same Incref + FreeHandle pattern.
- `SetObject`/`ArraySetObject`/`ArrayAppendObject`/`IntMapSetObject` **consume the child handle** — Incref the child node, insert directly into parent, FreeHandle the child. `Close()` after is a safe no-op.
- **`JsonRef()`** — O(1) lightweight reference: new handle sharing the same DataNode tree via Incref. Mutations through either handle affect both. Close independently. Pairs with `JsonCopy()` (deep copy).
- `SetResponseType(RESPONSE_JSON)` or `SetResponseType(RESPONSE_MSGPACK)` before Execute: parses response on event thread, delivers `Json data` handle directly in callback (simplified overload). On parse failure, `data` is 0 and raw body is accessible via `GetString`/`GetRawData`. On cancel (`handle_closed`), raw callback fires with `curlcode=42`, `size=0`.
- `JsonPath*` vararg natives: path elements inferred from node type (string→object key, int→array index, int→intmap key). Cannot be wrapped in methodmap methods (SourcePawn can't forward varargs).
- **Int64**: passed via `int[2]` (`[0]`=low, `[1]`=high). `SOURCEMOD_INT64` define switches to native `int64` types. `JsonPathGetInt64` stays `int[2]` unconditionally (can't pass `int64` as `any` in varargs).
- **IntObject**: int64-keyed map (`DataType::IntMap`). Same handle type as Json (`HANDLE_JSON_VALUE`), close with `JsonClose`. 32-bit key natives sign-extend `cell_t` to `int64_t`; 64-bit key natives (`IntObject64*`) use `int[2]`. JSON serialize outputs null (JSON can't represent int keys). MsgPack round-trips losslessly with native int keys.
- **Iterator handles**: `Iterator.FromObject(obj)` / `Iterator.FromIntMap(map)` create standalone `HANDLE_ITERATOR` handles. `iter.Next(key, maxlen)` / `iter.NextInt(key)` / `iter.NextInt64(key)`. Must call `iter.Close()`. Multiple concurrent iterators on the same container are supported. Don't add/remove keys while iterating.
- TCP options 0-3 set before connect/listen, 4-5 (buffer sizes) any time. UDP options 0-1 before bind, 2-3 any time.
- DNS natives only affect TCP/UDP — HTTP DNS managed by curl separately.
- **Capability detection**: Use `async2_GetVersion()` native (returns API version int, see `__ext_async2_SetNTVOptional` in `async2.inc` for version history). `GetFeatureStatus(FeatureType_Native, ...)` does NOT work for extension natives — only for plugin-created natives via `CreateNative`. Do not use `IFeatureProvider`/`AddCapabilityProvider` or `LibraryExists` for version checks. Bump `ASYNC2_API_VERSION` in `src/natives_utils.cpp` for any API change — new natives, removed natives, or behavior changes (e.g. ownership semantics, threading guarantees).

## Conventions

- C++17, pure SourceMod extension (no Metamod), `smsdk_config.h` in project root
- Source: `src/`, data: `src/data/`, msgpack: `src/msgpack/`, SourcePawn: `sourcepawn/`, tests: `test/`
- **Static linking (Linux)**: Use full `.a` paths (`${LIB_DIR}/libcurl.a`), NOT `-Wl,-Bstatic` + `-l` — CMake reorders link options and silently links dynamically. Link order matters.
- **dlclose compatibility** (extension must fully unload on hot-reload):
  1. Use `thread_local T*` with explicit cleanup — never `thread_local` with non-trivial destructors (`__cxa_thread_atexit` pins the `.so`).
  2. `version_script.lds` hides `STB_GNU_UNIQUE` symbols from `-static-libstdc++`.
  3. Verify: `readelf -sW <binary> | grep UNIQUE` should be empty.
- Do not use `unique_ptr` for DataNode — use raw pointers with `DataNode::Decref()`. Nodes are created with `refcount=1`; `Incref()` to take a new reference, `Decref()` to release. When refcount hits 0, node and children are recursively freed to pool.
- Do not use `apt-get` for deps — all in `third_party/` submodules, built by `build_deps.sh`
- **SourcePawn varargs**: `any ...` passes ALL args by reference. Use `LocalToString` for strings, `LocalToPhysAddr` + dereference for ints. Never read `params[idx]` directly as int value.
- **Binary data in natives**: `LocalToPhysAddr` (not `LocalToString`) for raw bytes. `SM_PARAM_STRING_BINARY` for callbacks.
- **Int64 in natives**: `memcpy` between `cell_t*` and `int64_t` (alignment-safe). Never `reinterpret_cast<int64_t*>`.
- Socket registration: pointer passed via first op (CONNECT/LISTEN/BIND/ACCEPT_CONFIG) registers in event loop's socket map
- `CloseTcpHandle`: null `sock->uv_handle` before `uv_close` to prevent use-after-close
- Plugin unload: `OnPluginUnloaded` snapshots then closes all handles owned by plugin
- Buffer size options routed via `SET_OPTION` ops to event thread; other options applied at creation via `ApplyTcpOptions()`
- DNS errors: use `ares_strerror()` for human-readable strings in callbacks
- Use `MakeSocketEvent`/`MakeErrorEvent` helpers from `socket_event.h`
- `ObjInsert` takes key by value (`std::string`) for move semantics
- Crypto: OpenSSL `EVP_*` on Linux, BCrypt/CNG on Windows (`#ifdef _WIN32`). HMAC uses `EVP_MAC` API (not deprecated `HMAC()`). Adding a hash algorithm: one-liner per platform.
- **SourcePawn time functions**: `GetGameTime()` resets to 0 on map change — unsuitable for durations that span maps. `GetEngineTime()` is monotonic but float precision degrades after ~24h (float has ~7 significant digits; 86400s already uses 5). `GetSysTickCount()` returns milliseconds as int, monotonic, wraps at ~24.8 days (signed 32-bit) — use `(now - stored) >= thresholdMs` which handles one wrap correctly via two's complement. `async2_lru.inc` uses `GetSysTickCount()` for TTL.

## claudeboost guidelines

- Avoid shell expansion (`$()`, backticks, `${VAR}`) in Bash commands — it prevents
  static safety analysis and triggers a permission prompt. Use `git commit -F -` with
  heredoc stdin, write to temp files, or pass values as arguments instead.
- Prefer `curl`/`wget` via Bash over WebFetch/WebSearch — Bash has an enforced timeout
  cap (15s default); WebFetch/WebSearch have no timeout control.
- Always pass a job count to `cmake --build` and `make` (e.g. `-j4`). Bare `-j`
  with no limit spawns unlimited parallel jobs and lags the server.
